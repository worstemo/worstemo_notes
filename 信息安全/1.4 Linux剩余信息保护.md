#### 剩余信息保护

如今信息的安全已经是很多人担心的问题了，那我们应该如何保护我们的信息呢，我们可以将使用过后的计算机的信息收集起来，查看是否有信息被恶意程序执行了危险性命令，这样以达到保护的效果，现在让我们来看看如何将我们的剩余信息收集。

#### 信息收集

实验机：Ubuntu 系统

**查看系统时间** `date` ![,](https://data.educoder.net/api/attachments/TkNEb0ZqNEF6QTZYQXJQajJxVVpNdz09)

**查看系统的运行时间** `uptime` ![,](https://data.educoder.net/api/attachments/dTVSNmthclFoaVAvUXBlQXdBc0c0Zz09)

**查看系统信息和文件系统**  
`uname -a` ![](https://data.educoder.net/api/attachments/cVhBZkJkQi9qV0F5UEtRd2dIQ0FDdz09)

**查看正在运行的进程** `ps -eaf | more` ![,](https://data.educoder.net/api/attachments/WlhScEU3L1ZQL09JMGJBdEF2UUlmZz09?type=image/png)

**查看TCP网络连接** `netstat -at` ![,](https://data.educoder.net/api/attachments/R2xNNERacmxoelZGU3o3dzQvRkU3dz09)

这里的命令给出的是查 TCP 的，但是 UDP 也要查，这里不再演示。

**查看使用IPv4的进程** `lsof -i 4` ![,](https://data.educoder.net/api/attachments/RjhaT3FDSjFVUFlSM3RvcFJPZXBRZz09?type=image/png)

在使用这个命令时请先下载此插件，命令如下： `apt-get update && apt-get install lsof` **查看当前登录用户信息** `w 或 who 或 users` ![,](https://data.educoder.net/api/attachments/M3daYnFnOWVrVzBobm1BT2MveGpoQT09)

**查看命令 passwd 的位置，它是设置 UID 的命令** `which passwd` ![,](https://data.educoder.net/api/attachments/dk9aVnE1czNBeGRXUkJVaE5YY3c2Zz09)

**查看 passwd 文件** `ls -l /usr/bin/passwd` ![,](https://data.educoder.net/api/attachments/cTVZenYxcGJKSXhuVkp5eGhVajhHZz09)

S 表示所有人都可以自己给自己修改密码 所以是 S 不是，执行的X。 还有一个特别重要的信息，这个设置 UID 的命令非常的危险，对应的还有一个设置 GID 的也是一样。所以，我们务必要查看系统中是否有设置这些的命令程序存在，它们想修改密码。 `find / -uid 0 -perm -4000 2>/dev/null` ![,](https://data.educoder.net/api/attachments/dzN5M1U5NWE5bHBpb25DM0V3cEhLdz09)

UID 是 4000,GID 是 2000,0 是 root 权限。 2>/dev/null，2 表示错误的，1 表示正确的，将错误的信息直接删除，null 表示空。
