**AWS Identity and Access Management (IAM)** 是一种 Web 服务，可支持 Amazon Web Services (AWS) 客户在 AWS 中管理用户和用户权限。借助 IAM，您可以集中管理**用户**、**安全凭证**（如访问密钥）以及**权限**（控制用户可以访问哪些 AWS 资源）。

## AWS Identity and Access Management

AWS Identity and Access Management (IAM) 可用于：

- **管理 IAM 用户及其访问权限**：您可以创建用户，并为其分配单独的安全凭证（访问密钥、密码和多重身份验证设备）。您可以管理权限，控制用户可以执行哪些操作。
    
- **管理 IAM 角色及其权限**：IAM 角色类似于用户，是一种附带权限策略的 AWS 身份，其中权限策略决定此身份在 AWS 中可以执行及不可以执行的操作。不过，角色并非唯一关联到某个人，而是供任何有需要的人_代入_。
    
- **管理联合身份用户及其权限**：您可以启用_联合身份_，从而允许企业中的现有用户访问 AWS 管理控制台、调用 AWS API 和访问资源，而无需为每个身份创建 IAM 用户。

6. 选择 **user-1** 链接。

7. 请注意，user-1 没有任何权限。
    
8. 选择 **Groups**（组）选项卡。
    
    user-1 也不是任何组的成员。
    
9. 选择 **Security credentials**（安全凭证）选项卡。
    
    user-1 分配有一个 **Console password**（控制台密码）。
    
10. 在左侧导航窗格中，选择 **User groups**（用户组）。
    
    已为您创建以下组：
    
    - EC2-Admin
        
    - EC2-Support
        
    - S3-Support
        
11. 选择 **EC2-Support** 组链接。
    
    此时会转到 **EC2-Support** 组的摘要页面。
    
12. 选择 **Permissions**（权限）选项卡。
    
    此组有一项与之关联的 **Managed Policy**（托管策略），称为 **AmazonEC2ReadOnlyAccess**。托管策略是可以关联到 IAM 用户和组的预构建策略（由 AWS 或您的管理员构建）。如果您更新了该策略，此更改将立即应用于策略关联到的所有用户和组。
    

13. 选择 AmazonEC2ReadOnlyAccess 策略旁边的加号 (**+**) 图标，查看策略详细信息。
    
    **注意**：策略定义允许或拒绝对特定 AWS 资源执行的操作。
    
    此策略授予的权限是，列出和描述有关 EC2、Elastic Load Balancing、CloudWatch 和 Auto Scaling 的信息。这种能够查看资源但不能修改资源的权限非常适合分配给支持角色。
    
    IAM 策略声明的基本结构是：
    
    - **Effect**（效果）：表示是 _Allow_（允许）还是 _Deny_（拒绝）权限。
        
    - **Action**（操作）：指定可以对 AWS 服务执行的 API 调用（例如 _cloudwatch:ListMetrics_）。
        
    - **Resource**（资源）：定义策略规则涵盖的实体范围（例如，特定的 Amazon S3 存储桶或 Amazon EC2 实例，还可以用 * 来表示_任何资源_）。
        
14. 选择减号 (**-**) 图标隐藏策略详细信息。
    

15. 在左侧导航窗格中，选择 **User groups**（用户组）。
    

16. 选择 **S3-Support** 组链接，然后选择 **Permissions**（权限）选项卡。
    
    S3-Support 组关联了 **AmazonS3ReadOnlyAccess** 策略。
    

17. 选择加号 (**+**) 图标查看策略详细信息。
    
    此策略授予的权限是，获取和列出 Amazon S3 中的资源。
    

18. 选择减号 (**-**) 图标隐藏策略详细信息。
    

19. 在左侧导航窗格中，选择 **User groups**（用户组）。
    

20. 选择 **EC2-Admin** 组链接，然后选择 **Permissions**（权限）选项卡。
    
    此组与其他两个组略有不同。它使用的不是 _Managed Policy_（托管策略），而是 **Inline Policy**（内联策略）。内联策略是一种仅分配给一个用户或组的策略，这种策略通常用于针对一次性情况应用权限。
    

21. 选择加号 (**+**) 图标查看策略详细信息。
    
    此策略授予查看（描述）有关 Amazon EC2 信息的权限，以及启动和停止实例的权限。
    

22. 选择减号 (**-**) 图标隐藏策略详细信息。
