## 实验概览与目标

本实验旨在强化利用 AWS 托管数据库实例满足关系数据库需求的概念。

_**Amazon Relational Database Service**_ (Amazon RDS) 可让您在云中轻松设置、运维和扩展关系数据库。它可在承担耗时的数据库管理任务的同时，提供经济高效的可调容量，使您能够腾出时间专注于应用程序和业务。Amazon RDS 提供了 6 个熟悉的数据库引擎供您选择：Amazon Aurora、Oracle、Microsoft SQL Server、PostgreSQL、MySQL 和 MariaDB。

本实验结束时，您将能够：

- 启动高度可用的 Amazon RDS 数据库实例。
- 将数据库实例配置为允许来自 Web 服务器的连接。
- 打开 Web 应用程序并与数据库交互。

## 任务 1：为 RDS 数据库实例创建安全组

在本任务中，您将创建一个安全组，以支持 Web 服务器访问您的 RDS 数据库实例。您将在启动数据库实例时使用此安全组。

4. 在 AWS 管理控制台中 Services（服务）旁边的搜索框中，搜索并选择 **VPC**。
    
5. 在左侧导航窗格中，选择 **Security Groups**（安全组）。
    
6. 选择 Create security group（创建安全组），然后进行以下配置：
    
    - **Security group name**（安全组名称）：`DB Security Group`
        
    - **Description**（描述）：`Permit access from Web Security Group`
        
    - **VPC**：_Lab VPC_
        
        **提示**：选择已选中 VPC 旁边的 X，然后从菜单中选择 **Lab VPC**。
        
    
7. 在 **Inbound rules**（入站规则）窗格中，选择 Add rule（添加规则）
    
    此安全组当前没有规则。您将添加一个规则，允许来自 _Web Security Group_ 的访问。
    
8. 配置以下设置：
    
    - **Type**（类型）：_MySQL/Aurora (3306)_
    - **Source**（源）：将光标放在 **Custom**（自定义）右边的字段中，键入 `sg`，然后选择 _Web Security Group_。
    
    这会配置数据库安全组，在端口 3306 上允许来自与 _Web Security Group_ 关联的任何 EC2 实例的入站流量。
    
9. 选择 Create security group（创建安全组）
    
    在本实验中，您将在启动 Amazon RDS 数据库时使用此安全组。
    

## 任务 2：创建数据库子网组

在本任务中，您将创建_数据库子网组_，用于指示 RDS 可以将哪些子网用于数据库。每个数据库子网组需要至少位于两个可用区中的子网。

10. 在 AWS 管理控制台中 Services（服务）旁边的搜索框中，搜索并选择 **RDS**。
    
11. 在左侧导航窗格中，选择 **Subnet groups**（子网组）。
    
    如果没有显示导航窗格，请选择左上角的 菜单图标。
    
12. 选择 Create DB Subnet Group（创建数据库子网组），然后进行以下配置：
    
    - **Name**（名称）：`DB-Subnet-Group`
        
    - **Description**（描述）：`DB Subnet Group`
        
    - **VPC**：_Lab VPC_
        
13. 向下滚动到 **Add Subnets**（添加子网）部分。
    
14. 展开 **Availability Zones**（可用区）下面的值列表，然后选择前两个区域：**us-east-1a** 和 **us-east-1b**。
    
15. 展开 **Subnets**（子网）下面的值列表，然后选择与 CIDR 范围 **10.0.1.0/24** 和 **10.0.3.0/24** 关联的子网。
    
    这些子网现在应显示在 **Subnets selected**（选中的子网）表中。
    
16. 选择 Create（创建）
    
    在下一个任务中创建数据库时，您将使用此数据库子网组。
    

## 任务 3：创建 Amazon RDS 数据库实例

在本任务中，您将配置并启动 MySQL 数据库实例的多可用区 Amazon RDS 部署。

Amazon RDS _**多可用区**_部署可以为数据库 (DB) 实例提供增强的可用性和持久性，从而使其成为生产数据库工作负载的理想之选。在您预置多可用区数据库实例时，Amazon RDS 会自动创建一个主数据库实例，并将数据同步复制到另一个可用区 (AZ) 中的备用实例。

17. 在左侧导航窗格中，选择 **Databases**（数据库）。
    
18. 选择 Create database（创建数据库）。
    
    如果您在屏幕顶部看到 **Switch to the new database creation flow**（切换到新数据库创建流程），请选择此选项。
    
19. 在 **Engine options**（引擎选项）下，选择 **MySQL**。
    
20. 在 **Templates**（模板）下，选择 **Dev/Test**（开发/测试）。
    
21. 在 **Availability and durability**（可用性与持久性）下，选择 **Multi-AZ DB instance**（多可用区数据库实例）。
    
22. 在 **Settings**（设置）下，进行以下配置：
    
    - **DB instance identifier**（数据库实例标识符）：`lab-db`
        
    - **Master username**（主用户名）：`main`
        
    - **Master password**（主密码）：`lab-password`
        
    - **Confirm password**（确认密码）：`lab-password`
        
23. 在 **DB instance class**（数据库实例类）下，进行以下配置：
    
    - 选择 **Burstable classes (includes t classes)**（可突增类（包括 t 类））。
        
    - 选择 _db.t3.micro_
        
24. 在 **Storage**（存储）下，进行以下配置：
    
    - **Storage type**（存储类型）：_General Purpose (SSD)_（通用型 (SSD)）
        
    - **Allocated storage**（分配的存储空间）：_20_
        
25. 在 **Connectivity**（连接）下，进行以下配置：
    
    - **Virtual Private Cloud (VPC)**：_Lab VPC_
        
26. 在 **Existing VPC security groups**（现有 VPC 安全组）下，从下拉列表中：
    
    - 选择 _DB Security Group_。
        
    - 取消选中 _default_（默认）。
        
27. 在 **Monitoring**（监控）下：
    
    - _取消选中_ **Enable Enhanced monitoring**（启用增强监控）。
        
28. 在页面底部，展开 **Additional configuration**（其他配置），然后进行以下配置：
    
    - **Initial database name**（初始数据库名称）：`lab`
        
    - _取消选中_ **Enable automatic backups**（启用自动备份）。
        
        这会禁用备份，通常不建议这样做，但在本实验中可以更快地部署数据库。
        
    - _取消选中_ **Enable encryption**（启用加密）
        
    
29. 选择 Create database（创建数据库）。
    
    您的数据库将立即启动。
    
    如果您收到的错误消息提到 **not authorized to perform: iam:CreateRole**（没有权限执行：iam:CreateRole），请确保在上一步中取消选中了 _Enable Enhanced monitoring_（启用增强监控）。
    
30. 选择 **lab-db**（选择链接本身）。
    
    现在，您需要等待**大约 4 分钟**才能使用数据库。部署过程在两个不同的可用区中部署数据库。
    
    在等待期间，您可以查看 [Amazon RDS 常见问题解答](https://aws.amazon.com/rds/faqs/)，或者稍作休息。
    
31. 等到 **Info**（信息）变为 **Modifying**（正在修改）或 **Available**（可用）。
    
32. 向下滚动到 **Connectivity & security**（连接和安全性）部分，并复制 **Endpoint**（终端节点）字段。
    
    该值类似于：_lab-db.xxxx.us-east-1.rds.amazonaws.com_。
    
33. 将终端节点的值粘贴到文本编辑器中。您稍后将在实验中使用该值。
    

## 任务 4：与数据库交互

在本任务中，您将打开一个运行在已创建好的 Web 服务器上的 Web 应用程序。您将配置该应用程序，让它使用您刚才创建的数据库。

34. 要找到 **WebServer** 的 IP 地址，请选择本说明上方的 **AWS Details**（AWS 详细信息）下拉菜单。复制 IP 地址值。
    
35. 打开新的 Web 浏览器标签页，粘贴 _WebServer_ IP 地址，然后按 Enter 键。
    
    此时将显示 Web 应用程序，其中显示有关 EC2 实例的信息。
    
36. 选择页面顶部的 **RDS** 链接。
    
    现在，您需要将应用程序配置为连接到您的数据库。
    
37. 配置以下设置：
    
    - **Endpoint**（终端节点）：粘贴您之前复制到文本编辑器的终端节点
    - **Database**（数据库）：`lab`
    - **Username**（用户名）：`main`
    - **Password**（密码）：`lab-password`
    - 选择 **Submit**（提交）。
    
    此时将显示一条消息，说明应用程序正在运行命令，将信息复制到数据库。在几秒钟后，应用程序将显示 **Address Book**（地址簿）。
    
    地址簿应用程序使用 RDS 数据库存储信息。
    
38. 通过添加、编辑和删除联系人来测试 Web 应用程序。
    
    数据将持久保存在数据库中，并自动复制到第二个可用区。
